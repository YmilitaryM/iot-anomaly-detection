第一段：检测管线设计

数据从进入到输出异常事件的完整流转

Pipeline 四阶段

检测管线 — 逐阶段流转
⏱ Stage 1: 预处理 & 时序分解
原始数据点 → 缺失值填充(前向填充+线性插值) → STL 分解 → 输出: 残差分量 + 趋势分量 + 季节分量
关键决策：STL 比 Prophet 更适合传感器高频数据，鲁棒性更好，不需要手动调周期
🔧 Stage 2: 多维特征工程
从残差序列计算: 滑动窗口均值/方差/偏度/峰度、自相关、频谱熵、差分统计量
跨传感器: 相关系数矩阵、互信息、Granger因果(可选)
关键决策：特征计算在 Flink 窗口内完成，保证实时性；跨传感器特征异步更新，延迟可接受
🎯 Stage 3: 多策略并行检测
每个检测器独立运行，输出 (anomaly_score: 0~1, confidence: 0~1, evidence: str)

┌──────────────────────────────────────────────────┐
│ 检测器1 统计规则 ──→ score₁ │
│ 检测器2 时序残差 ──→ score₂ │
│ 检测器3 邻近度 ────→ score₃ │ → 融合器 → 最终异常判定
│ 检测器4 预测偏差 ──→ score₄ │
│ 检测器5 变点检测 ──→ score₅ │
└──────────────────────────────────────────────────┘
 关键决策：每个检测器可独立开关和配置，新传感器类型可自选启用哪些检测器
📊 Stage 4: 融合 & 裁决
weighted_score = Σ(scoreᵢ × weightᵢ × confidenceᵢ) / Σ(weightᵢ)
初始权重均等(各0.2)，随反馈调整
阈值策略: weighted_score > 0.7 → 告警 | 0.4~0.7 → 疑似(记录不告警) | < 0.4 → 正常
抑制策略: 同一设备同一传感器 5 分钟内去重合并 关键决策：阈值分级策略防止告警风暴；去重窗口避免同一异常反复告警

检测器配置策略（按传感器类型）

传感器类型 统计 残差 邻近度 预测 变点 说明
振动(高频) ✅✅✅✅✅ 全开，频谱特征权重高
温度/湿度(低频) ✅✅—✅✅ 邻近度对缓变数据意义不大
开关量/状态量 ✅———✅ 主要是规则+状态变化检测
电流/功率(中频) ✅✅✅✅✅ 全开，多维关联检测重要
