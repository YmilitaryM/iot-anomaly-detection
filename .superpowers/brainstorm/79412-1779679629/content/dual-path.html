解决方案：双路并行 + 鲁棒归一化

不让条件归一化变成"异常橡皮擦"

核心思路：不替代，而是叠加

双路并行的检测架构
🛤 Path A: 全局检测
输入: 原始传感器值 (raw value)
不经过条件归一化
统计规则使用物理/安全阈值
职责: 守住硬边界

例: 不论什么工况，温度超过120°C
就是红线，必须告警
🛤 Path B: 上下文检测
输入: 条件归一化后的值
经过上下文感知层处理
使用鲁棒统计算子(MAD替代Std)
职责: 发现隐式异常

例: 重载时温度85°C，没超硬边界
但同工况其他设备才70°C，这台异常
Path A 和 Path B 各自跑完5个检测器 → 两路分数独立计算 → 进入融合器时取 max(pathA_score, pathB_score)
融合策略:
final_score = max(global_score, contextual_score)
→ 任何一路认为异常，就触发

但告警分级不同:
• global 触发 → 等级: CRITICAL (物理边界突破)
• contextual 触发 → 等级: WARNING (模式内异常)
• 两者同时触发 → 等级: CRITICAL + 高置信

附加保护措施

保护措施 解决的问题 实现方式
鲁棒统计算子 模式内部离群点拉高标准差，导致归一化后异常不明显 用 MAD (Median Absolute Deviation) 替代 Std
MAD = median(|xᵢ - median(x)|)
MAD 对离群点的敏感度比 Std 低 50%+
模式置信度门控 工况切换过渡期的数据，分不清属于哪个模式 mode_confidence < 0.7 时，Path B 输出置为 0
过渡态只走 Path A 全局检测
新簇异常标记 DBSCAN 把异常点聚成自己的"模式" 新簇点数 < min_samples × 3 → 不认定为新模式
标记为异常簇，直接触发告警
硬边界永远优先 任何归一化都不应覆盖物理/安全边界 每个传感器可配置 absolute_min / absolute_max
硬边界突破直接 CRITICAL，短路跳过所有检测器

更新后的完整 Pipeline

⚡ 硬边界检查 (短路层)
Stage 1: 预处理 + STL
Stage 1.5: 上下文感知层
Stage 2: 特征工程
Path A: 全局检测 (原始值)
Path B: 上下文检测 (归一化值)
Stage 3: 5检测器 × 2路 = 10个分数
Stage 4: max融合 + 分级告警
